# 简介区域样式偏左问题修复说明

## 问题描述
简介区域（AboutSection）的内容没有正确居中，而是偏向左边。

## 问题原因
1. **样式优先级问题**：`<style scoped>` 会为每个选择器添加 data 属性，可能导致 Tailwind CSS 的优先级问题
2. **重复的样式定义**：在多个文件中重复定义了相同的样式
3. **Tailwind CSS 引入方式**：需要确保正确引入 Tailwind CSS

## 修复方案

### 1. 修改 `src/styles/main.css`
- 添加 `@import "tailwindcss"` 正确引入 Tailwind CSS
- 添加 `body` 样式确保全局样式正确
- 添加 `.bg-dots` 类（参考文件中的点状背景）
- 移除手动定义的 Tailwind 类（避免冲突）

### 2. 修改 `src/App.vue`
- 更新 template 结构，使用正确的类
- 移除重复的样式定义
- 添加 `antialiased bg-dots selection:bg-pink-200 selection:text-pink-900` 类
- **页脚始终在底部**：使用 flexbox 布局，`min-h-screen flex flex-col`，主内容区域 `flex-1`
- **页面切换时滚动到顶部**：使用 `router.afterEach` 钩子

### 3. 修改 `src/views/AboutSection.vue`
- 将 `<style scoped>` 改为 `<style>`，避免 scoped 样式导致的优先级问题

## 关键修改点

### App.vue - 页脚始终在底部
```html
<template>
  <div class="relative min-h-screen flex flex-col antialiased bg-dots selection:bg-pink-200 selection:text-pink-900">
    <!-- 背景粒子画布 -->
    <canvas ref="canvasRef" class="fixed inset-0 z-0 pointer-events-none w-full h-full"></canvas>

    <!-- 主要内容区域 -->
    <div class="relative z-10 flex flex-col min-h-screen">
      <Navbar />

      <!-- 主内容（自动撑满剩余空间） -->
      <main class="flex-1">
        <router-view v-slot="{ Component }">
          <transition name="page-fade" mode="out-in">
            <component :is="Component" />
          </transition>
        </router-view>
      </main>

      <!-- 页脚始终在底部 -->
      <FooterSection />
    </div>
  </div>
</template>
```

### App.vue - 页面切换时滚动到顶部
```javascript
import { useRouter } from 'vue-router'

const router = useRouter()

// 页面切换时滚动到顶部
router.afterEach(() => {
  window.scrollTo({ top: 0, behavior: 'smooth' })
})
```

## 验证方法
1. 运行 `npm run dev` 启动开发服务器
2. 访问页面，检查简介区域是否正确居中
3. 切换页面，检查是否自动滚动到顶部
4. 检查页脚是否始终在页面底部

## 注意事项
- Tailwind CSS v4 使用 `@import "tailwindcss"` 引入
- 避免在多个文件中重复定义相同的样式
- 使用 `<style>` 而不是 `<style scoped>` 来避免优先级问题
- 使用 `router.afterEach` 钩子实现页面切换时的自动滚动
